DEFINITION OF AN ANY PENNIE

An Any Pennie (commonly referred to as an "Any") is formally defined as a binding obligation requiring the sequential and uninterrupted consumption of the following items:

  • Primary Consumption: The complete consumption of one (1) 0.5L can of Grolsch Kanon (11.6% ABV).
  • Subsequent Consumption: Immediately following the primary consumption, the consumption of one (1) traditional borrel (not a shot!) of Ketel 1 ambachtelijke jonge graanjenever (35% ABV).

Note: Both conditions must be fulfilled in this exact chronological order within a proper timeframe. Failure to comply invalidates the redemption, and the outstanding balance remains unchanged in the official Treasury record.

THE LAWS OF THE TREASURY

01

The Assignment Clause

Whenever a Treasurer receives an any on someone, it must be logged against that person's name before the ledger is considered complete.

02

The Redemption Clause

When a person takes an any, the committee records the action immediately so the remaining count stays accurate.

03

Auditor Immunity

Only designated auditors can add or close an anytimer entry. The person owed the any does not self-approve their own balance.

04

The Sovereign Balance

The dashboard is always driven by the remaining any count per person. The audit trail is the source of truth, and the cards are only the current summary.

ABOUT THE SCOREBOARD

Thalia's Treasurer's Anytimer Scoreboard

This system was custom-built by and for Thalia's (old) Treasurers, and serves as an auditable record of who still has anys left to take. It keeps a person-by-person balance and an immutable audit trail for each assignment and redemption.

Any Received

A Treasurer logs that someone has been assigned one or more anys. They will then update the scoreboard.

Any Taken

A Treasurer logs when the owed any is properly fulfilled. The any should be logged as soon as it's taken.

Technical Stack

Built with native HTML5, CSS3, and JavaScript, powered by Vercel Serverless API functions and Vercel KV for global state management. Secures edits via HMAC SHA-256 session signatures.
